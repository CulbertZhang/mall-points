> 设计版本：v1.0 | 日期：2026-06-07
> 参考原型：有赞微商城（shop93128405.m.youzan.com）

---

## 一、项目概述

本项目旨在设计并实现一个完整的商城会员积分系统，核心目标是构建 **"积分获取 → 积分消耗 → 复购转化"** 的一站式会员运营闭环。系统参考有赞微商城的完整功能体系，覆盖首页、分类、购物车、会员中心四大核心场景，并以积分体系为纽带串联会员等级、营销活动和交易全链路。

### 系统能力矩阵

| 能力域 | 覆盖范围 |
|--------|----------|
| 积分获取 | 消费返积分、每日签到、任务中心、大转盘抽奖、邀请好友 |
| 积分消耗 | 兑换优惠券、兑换实物商品、积分+现金组合支付、积分抵现、积分抽奖 |
| 会员等级 | VIP1-VIP10 十级体系、成长值引擎、积分倍率、等级权益、升降级规则 |
| 营销工具 | 优惠券、限时秒杀、拼团、大转盘抽奖、公告栏 |
| 交易链路 | 商品浏览→加购→下单→支付→收货→评价→积分到账 |
| 数据运营 | 积分发放/消耗分析、人群分层、自动化触达 |

## 四、核心业务流程

### 4.1 积分获取流程

```
                     ┌─────────────┐
                     │  用户行为    │
                     └──────┬──────┘
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
    ┌─────────┐       ┌──────────┐        ┌──────────┐
    │ 消费下单 │       │ 每日签到  │        │ 完成任务  │
    │ 1元=1积分│       │ 连续递增  │        │ 浏览/分享 │
    └────┬────┘       └────┬─────┘        └────┬─────┘
         │                 │                   │
         ▼                 ▼                   ▼
    ┌─────────────────────────────────────────────┐
    │              积分计算引擎                     │
    │  • 应用会员等级倍率 (VIP2×1.2, VIP3×1.5...)  │
    │  • 校验日/周/月上限                          │
    │  • 防刷风控检查                               │
    └────────────────────┬────────────────────────┘
                         ▼
    ┌─────────────────────────────────────────────┐
    │  points_transactions 写入流水                │
    │  points_accounts.balance += amount          │
    │  乐观锁 version 并发控制                     │
    └────────────────────┬────────────────────────┘
                         ▼
    ┌─────────────────────────────────────────────┐
    │  通知推送 (积分变动提醒)                      │
    │  前端轮询/WebSocket 实时更新积分余额          │
    └─────────────────────────────────────────────┘
```

### 4.2 积分消耗流程（积分抵现）

```
    用户进入确认订单页
           │
           ▼
    GET /api/v1/order/preview  ← 返回可用积分、抵扣汇率、最大抵扣比例
           │
           ▼
    用户勾选"使用积分抵扣"，输入抵扣金额
           │
           ▼
    POST /api/v1/orders  ← { pointsDeduction: 1000, ... }
           │
           ▼
    服务端校验:
    • 积分余额 ≥ 抵扣金额 × 汇率 (100积分=1元)
    • 抵扣金额 ≤ 订单金额 × 50%
           │
           ▼
    创建订单 + 锁定积分(frozen_points)
    支付成功后: 正式扣除积分 + 按实付金额返积分
    支付失败/取消: 解冻积分
```

### 4.3 会员等级升降级流程

```
    ┌──────────────────────────────────────┐
    │         用户完成有效订单              │
    │  获得成长值 = 实付金额(元) × 规则     │
    └──────────────┬───────────────────────┘
                   ▼
    ┌──────────────────────────────────────┐
    │  growth_value_transactions 写入流水   │
    │  growth_value_accounts.balance += N  │
    └──────────────┬───────────────────────┘
                   ▼
    ┌──────────────────────────────────────┐
    │  判断是否需要升级                     │
    │  balance >= next_level.min_growth?   │
    └──────┬───────────────┬───────────────┘
          YES              NO
           │                │
           ▼                ▼
    ┌──────────────┐  ┌─────────────┐
    │ 升级到新等级  │  │ 维持当前等级 │
    │ 发放升级礼包  │  │             │
    │ (积分+券)    │  └─────────────┘
    │ 推送升级通知  │
    └──────────────┘

    定时任务 (每日凌晨):
    ┌──────────────────────────────────────┐
    │  检查降级条件                         │
    │  growth_value < current_level.min?   │
    │  AND 降级保护期已过?                  │
    └──────┬───────────────────────────────┘
          YES → 降级处理 + 推送降级通知
          NO  → 跳过
```

### 5.2 核心设计原则

- **乐观锁并发控制**: `points_accounts`、`growth_value_accounts`、`orders` 使用 `version` 字段
- **数据快照冗余**: 订单冗余收货地址快照；`order_items` 冗余商品名称、价格快照；`user_coupons` 冗余优惠券关键字段
- **分区策略**: `points_transactions`、`growth_value_transactions` 按月 RANGE 分区，保留 6 个月热数据
- **金额以分为单位**: 所有价格字段使用 `NUMERIC(12,2)` 或 `INTEGER`（分），避免浮点精度
- **软删除**: `users`、`products`、`user_addresses` 使用 `deleted_at` 字段

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| **高并发秒杀场景库存超卖** | 资金损失、用户投诉 | Redis 预减库存 + Lua 原子操作 + 异步落库；数据库乐观锁兜底 |
| **积分发放重复/漏发** | 客诉、对账异常 | 幂等性设计（唯一流水号）；消息队列 + 死信队列重试；每日对账脚本 |
| **积分被刷** | 积分通胀、运营成本失控 | 多维度风控（设备指纹/IP/账号频率限制）；异常行为实时预警 |
| **会员等级计算偏差** | 等级错乱影响权益 | 成长值流水可追溯；等级变更记录独立日志；离线重算对比校验 |
| **移动端兼容性** | iOS/Android 渲染异常 | 使用成熟 UI 库（Ant Design Mobile）；多机型真机测试 |
| **微信支付回调延迟** | 订单状态卡在待付款 | 主动轮询 + 回调双通道；超时自动关单；用户手动刷新支付状态 |
